# InRecord 開發流程

每次要新增功能或修改，照這份流程走就對了。

> ⚠️ 兩個重點先記住：
> 1. 本地測試一律用 **`http://127.0.0.1:3000`**（不要用 `localhost`，瀏覽器會有快取/連線問題）。
> 2. **Vercel 沒有連 GitHub 自動部署** → `git push` 之後，正式站要更新一定要再手動跑 `npx vercel --prod`。

---

## 一、本地開發（自己邊改邊看）

### 1. 啟動本地 server
```bash
npm run dev
```
啟動後開瀏覽器：**http://127.0.0.1:3000**

- Next.js 有熱更新：存檔後畫面會自動刷新，不用重開 server。
- 這階段只有你自己看得到，外面的人看不到。
- 要停止 server：在跑 `npm run dev` 的終端機按 `Ctrl + C`。

### 2. 改 code → 在 127.0.0.1:3000 邊改邊確認
直到畫面、功能都符合預期。

---

## 二、確認沒壞（上線前必做）

```bash
npm run build
```
看到 **`✓ Compiled successfully`**（exit code 0）才代表正式環境能正常 build。
如果這裡報錯，先修好再繼續，不要硬上線。

---

## 三、存檔到 GitHub

```bash
git add -A
git commit -m "簡短描述這次改了什麼"
git push
```

> push 只是把程式碼存到 GitHub，**不會**讓正式站更新。

---

## 四、部署到正式站（讓大家看到）

```bash
npx vercel --prod
```
- 約 30～60 秒完成，結束後會自動更新正式網址。
- 正式網址：**https://inrecord-swart.vercel.app**
- 這步**一定要手動跑**，因為 Vercel 沒連 GitHub。

## 常見問題

**Q：`localhost:3000` 打不開 / ERR_CONNECTION_REFUSED？**
A：改用 `http://127.0.0.1:3000`。你的瀏覽器對 `localhost` 有快取問題。

**Q：push 了，但正式站還是舊的？**
A：因為 Vercel 沒連 GitHub。要再跑 `npx vercel --prod` 才會更新。

**Q：金流（購買按鈕）在本地測沒反應？**
A：需要先在 Vercel 設定 PAYUNi 環境變數（PAYUNI_MERCHANT_ID / HASH_KEY / HASH_IV），刷卡開通才會真正生效。
